# 文档索引

本目录用于沉淀将当前“本地 CSV 看板”重构为“Git 仓库自动下载与分析服务”的设计文档。

## 文档列表

- `architecture.md`
  - 重构背景、目标架构、核心流程、存储策略和实施阶段。
- `data-model.md`
  - 项目、分支、分析任务、本地缓存和结果文件的数据模型设计。
- `api-design.md`
  - 后端服务的接口设计、请求响应结构和典型交互流程。
- `deployment.md`
  - 单机部署、端口、日志、启停脚本和日常运维方式。

## 当前结论

- 一个 Git 仓库对应一个项目。
- 一个项目下可配置多个分支作为分析目标。
- 项目创建和后续更新时都可以重新同步远端分支。
- 本地只保留一份仓库缓存，不为每个分支单独 clone。
- 分析结果以 JSON 为主存储格式，CSV 作为可选导出格式。
- 页面刷新时优先读取已保存结果，不依赖重新扫描本地代码。
