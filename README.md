# 研发效能深度分析看板 (R&D Efficiency Analytics Dashboard)

这是一个基于 React + TypeScript + Vite + ECharts 构建的纯前端、高性能的研发效能数据分析看板。它可以将散乱的 CSV 代码统计数据，转化为直观的多维分析图表，帮助团队领导者和开发者快速掌握团队产出动态。

## 🌟 核心特性

- **🚀 零配置一键启动**：自带 Windows 批处理启动脚本 (`start_dashboard.bat`)，一键安装依赖并启动服务。
- **🌐 局域网无缝共享**：服务启动后自动绑定 `0.0.0.0`，同局域网（如公司内网）的同事可直接通过 IP 地址免安装、免上传查看最新数据。
- **📂 自动目录感知**：默认自动扫描并加载本地 `hcc_simulate`（及其他同级文件夹）下的所有 `.csv` 文件，并且**自动将文件所在的直接父文件夹识别为“顶级工程”**。
- **👥 人员身份归一化**：支持通过修改 `author_mapping.json` 文件，自动将不同工号、别名、拼音的提交记录无缝合并到同一个真实姓名下。
- **🧹 智能脏数据清洗**：自带“显示/隐藏未知”数据开关，一键剔除没有签名、没有日期或解析异常的干扰数据。
- **📊 全维度数据下钻**：
  - **工程维度**：支持多工程的任意组合、合集统计与单选过滤。
  - **时间维度**：直观的月度产出堆叠趋势图。
  - **人员维度**：Top N 开发者产出横向柱状图，以及精准到个人的专属产出趋势折线图和详细流水账。
  - **指标灵活切换**：一键在“新增行数”、“删除行数”、“净增行数”中全局切换所有图表和表格的视角。

## 🛠️ 快速开始

### 前置条件
确保您的电脑上已安装 [Node.js](https://nodejs.org/) (推荐 v18 或更高版本)。

### Windows 推荐启动方式
双击项目根目录下的 **`start_dashboard.bat`**。
脚本将自动：
1. 检查 Node.js 环境。
2. 安装必要的项目依赖 (仅首次需要)。
3. 启动开发服务器并监听局域网 IP。
4. 自动在您的默认浏览器中打开看板。

### 通用命令行启动
如果您使用 macOS/Linux 或更喜欢命令行：
```bash
# 1. 安装依赖
npm install

# 2. 启动服务
npm run dev
```

## 📁 目录结构与数据准备

您不需要修改任何前端代码即可分析您的数据。只需按照以下约定放置您的 CSV 文件：

```text
根目录/
├── author_mapping.json   <-- (重要) 人员别名映射配置
├── hcc_simulate/         <-- 您的数据文件夹 A (名字将作为顶级工程名)
│   ├── 1.csv
│   └── 2.csv
├── hcc_backend/          <-- 您的数据文件夹 B
│   └── stats.csv
├── src/                  <-- 源码目录
└── start_dashboard.bat   <-- Windows 启动脚本
```

### 数据字段要求 (CSV)
看板默认解析以下列名（请确保导出的 CSV 包含这些列头）：
* `月份` (格式建议 YYYY-MM)
* `作者` (将被 `author_mapping.json` 翻译)
* `项目` (即子模块名)
* `新增行数`、`删除行数`、`净增行数`
* `有效Commit数`

## 🧑‍🤝‍🧑 人员身份归一化指南

为了解决同一位开发者使用不同工号或别名提交代码导致的数据分散问题，请编辑根目录下的 **`author_mapping.json`**：

```json
{
  "h30049752": "张三",
  "zhang_san": "张三",
  "j12345678": "李四",
  "lisi": "李四"
}
```
**配置后无需重启服务**，只需在浏览器中刷新页面，所有散乱的账号数据就会自动合并到对应的真实姓名名下。

直接将这个链接发送给连接同一公司 WiFi/内网的同事即可！他们打开链接后，将**直接看到您本地最新的数据和映射结果，完全不需要他们手动上传任何文件。**

 *(注：如果同事无法访问，请检查您的 Windows Defender 防火墙是否放行了 3000 端口的 TCP 协议)*
