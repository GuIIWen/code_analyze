# 研发效能分析系统设计文档 (R&D Efficiency Analytics Dashboard)

## 1. 系统背景与目标
针对当前多工程（CSV 文件形式）存储的研发统计数据，构建一个前端分析页面。该系统需解决跨工程、跨模块、跨人员的多维数据聚合问题，提供灵活的开发量统计与可视化看板。

### 核心指标
- **总开发量**：基于净增行数（Net Lines）的累加。
- **活跃度**：基于有效 Commit 数（Valid Commits）的统计。
- **时间维度**：按月度（Monthly）展示趋势变化。

---

## 2. 数据架构设计

### 2.1 数据源特征 (基于现有的 CSV 结构)
- **月份**：`YYYY-MM` 格式。
- **作者**：开发者唯一标识。
- **项目**：代表子工程或模块名（需过滤掉“总计”行）。
- **指标列**：新增行数、删除行数、净增行数、有效 Commit 数。

### 2.2 数据处理逻辑 (Data Engine)
1.  **文件聚合**：将多个 CSV 文件名映射为“工程组/一级工程”。
2.  **清洗逻辑**：
    - 自动过滤 `项目 == '总计'` 的冗余行，确保聚合统计的准确性。
    - 统一日期格式，便于跨文件排序。
3.  **多维索引**：构建 `{ 工程: { 模块: { 作者: [月度数据] } } }` 的树形索引结构。

---

## 3. 功能模块设计

### 3.1 概览看板 (Overview)
- **总览卡片**：展示选定范围内所有工程的总净增行数、总提交数、参与人数。
- **全工程月度趋势图**：折线图展示整体研发产出的起伏。

### 3.2 工程/模块分析 (Project View)
- **工程贡献占比**：饼图/环形图展示各工程在总开发量中的权重。
- **模块下钻**：选择特定工程后，展示其内部各子模块的开发比例。

### 3.3 个人效能分析 (Contributor View)
- **开发者排行榜**：按总产出或提交数排序。
- **个人项目分布**：展示特定人员在不同工程间的投入产出比。
- **个人月度贡献曲线**：分析人员的产出稳定性。

### 3.4 交互功能 (Interactive Features)
- **全局筛选器**：支持按时间段（Start-End Month）、工程组（文件名）、特定作者进行联动筛选。
- **数据透视表**：支持将数据导出为 Excel 或在网页端进行行列转置查看。

---

## 4. 技术栈推荐
- **框架**：React + TypeScript
- **UI 组件**：Ant Design (使用 Table, DatePicker, Select 等组件)
- **可视化库**：ECharts 或 Ant Design Charts
- **数据解析**：PapaParse (用于前端高性能解析大体积 CSV)

---

## 5. 实施路线图
1.  **Phase 1 (MVP)**：实现多 CSV 文件上传/加载，完成基础数据聚合及表格展示。
2.  **Phase 2 (Visual)**：集成 ECharts，实现趋势折线图和贡献度饼图。
3.  **Phase 3 (Deep Dive)**：增加多维筛选器，支持按人、按工程的深度钻取分析。
